# 贡献指南

感谢您对 LeetCode Hot 100 Solutions 项目的关注！

## 如何添加新题解

### 1. 选择合适的分类目录

在 `problems/` 目录下，根据问题的主要类型选择相应的分类文件夹：

- `array/` - 数组相关问题
- `string/` - 字符串相关问题
- `linked-list/` - 链表相关问题
- `tree/` - 树相关问题
- `dynamic-programming/` - 动态规划
- `backtracking/` - 回溯算法
- `stack/` - 栈相关问题
- `queue/` - 队列相关问题
- `heap/` - 堆相关问题
- `hash-table/` - 哈希表相关问题
- `two-pointers/` - 双指针技巧
- `sliding-window/` - 滑动窗口
- `greedy/` - 贪心算法
- `binary-search/` - 二分查找
- `graph/` - 图相关问题
- `dfs-bfs/` - 深度优先搜索/广度优先搜索
- `bit-manipulation/` - 位运算
- `math/` - 数学问题

### 2. 使用模板创建题解文件

参考项目根目录下的 `TEMPLATE.md` 文件，创建新的题解文件。

文件命名格式：`[问题编号]-[问题名称].md`

例如：`001-two-sum.md`

### 3. 填写题解内容

- **问题描述**：清晰描述问题，包括示例和约束条件
- **解法思路**：详细说明解题思路和关键点
- **代码实现**：提供多种语言的代码实现（Python、Java、C++ 等）
- **复杂度分析**：分析时间和空间复杂度
- **相关题目**：列出相关或类似的题目
- **标签**：添加相关标签便于检索

### 4. 更新主 README

在主 `README.md` 文件中的题目列表里添加新题目的链接。

### 5. 提交规范

- 确保代码格式规范，可以正常运行
- 提交信息清晰明了
- 一次提交专注于一个问题的解答

## 代码规范

### Python
- 遵循 PEP 8 规范
- 使用有意义的变量名
- 添加必要的注释

### Java
- 遵循 Java 命名规范
- 使用驼峰命名法
- 添加必要的注释

### C++
- 遵循 C++ 最佳实践
- 使用清晰的命名
- 添加必要的注释

## 质量标准

- 代码必须经过测试验证
- 解题思路清晰易懂
- 包含复杂度分析
- 格式规范统一
