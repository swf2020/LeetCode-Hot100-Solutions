# 406. 根据身高重建队列

[LeetCode 题目链接](https://leetcode.cn/problems/queue-reconstruction-by-height?envType=problem-list-v2&envId=2cktkvj)

##  问题简介

假设有打乱顺序的一群人站成一个队列，数组 `people` 表示队列中一些人的属性（不一定按顺序）。每个 `people[i] = [hi, ki]` 表示第 `i` 个人的身高为 `hi`，前面 **正好** 有 `ki` 个身高大于或等于 `hi` 的人。

请你重新构造并返回输入数组 `people` 所表示的队列。返回的队列应该格式化为数组 `queue`，其中 `queue[j] = [hj, kj]` 是队列中第 `j` 个人的属性，并且：

- 队列中排在第 `j` 个人前面的人数 **恰好** 为 `kj`；
- 满足上述条件的前提下，任意排列都是正确的。

### 示例说明

**示例 1：**

```
输入：people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
输出：[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
解释：
编号为 0 的人身高为 5 ，没有身高更高或者相同的人排在他前面。
编号为 1 的人身高为 7 ，没有身高更高或者相同的人排在他前面。
编号为 2 的人身高为 5 ，有 2 个身高更高或者相同的人排在他前面，即编号为 0 和 1 的人。
编号为 3 的人身高为 6 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
编号为 4 的人身高为 4 ，有 4 个身高更高或者相同的人排在他前面，即编号为 0、1、2、3 的人。
编号为 5 的人身高为 7 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
因此 [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]] 是重新构造后的队列。
```

**示例 2：**

```
输入：people = [[6,0],[5,0],[4,0],[3,2],[2,2],[1,4]]
输出：[[4,0],[5,0],[2,2],[3,2],[1,4],[6,0]]
```

## 💡 解题思路

这个问题的关键在于如何安排插入顺序，使得每个人都能被正确放置到满足其 `k` 值的位置。

### 思路一：贪心 + 排序 + 插入

1. **排序策略**：

   - 首先按照身高 **从高到低** 排序；
   - 如果身高相同，则按 `k` 值 **从小到大** 排序。

   这样做的原因是：高个子的人对矮个子的人没有影响（因为矮个子不会计入高个子的 `k` 值），但高个子的位置会影响矮个子的 `k` 值。所以我们先安排高个子。

1. **插入过程**：

   - 初始化一个空列表 `result`；
   - 遍历排序后的 `people`，将每个人插入到 `result` 的第 `k` 个位置；
   - 因为此时 `result` 中只有比当前人高或等高的人，所以插入到第 `k` 位就能保证前面恰好有 `k` 个 >= 当前身高的人。

### 思路二：使用线段树 / 树状数组（进阶）

> 此方法适用于大规模数据或需要更高效插入的场景，但在本题中由于数据规模较小，通常不必要。核心思想是维护一个“空位”数组，通过二分+树状数组快速找到第 `k+1` 个空位进行填充。此处略去详细展开。

## 📌 代码实现

<div class="code-tabs">

```java:Java
class Solution {
    public int[][] reconstructQueue(int[][] people) {
        // 排序：身高降序，k 升序
        Arrays.sort(people, (a, b) -> {
            if (a[0] != b[0]) {
                return b[0] - a[0]; // 身高高的在前
            }
            return a[1] - b[1];     // k 小的在前
        });

        List<int[]> result = new ArrayList<>();
        for (int[] p : people) {
            result.add(p[1], p); // 插入到第 k 个位置
        }

        return result.toArray(new int[people.length][]);
    }
}
```

```go:Go
func reconstructQueue(people [][]int) [][]int {
    // 排序：身高降序，k 升序
    sort.Slice(people, func(i, j int) bool {
        if people[i][0] != people[j][0] {
            return people[i][0] > people[j][0]
        }
        return people[i][1] < people[j][1]
    })

    result := make([][]int, 0, len(people))
    for _, p := range people {
        // 在 Go 中需手动插入
        pos := p[1]
        result = append(result, []int{}) // 扩容
        copy(result[pos+1:], result[pos:]) // 后移
        result[pos] = p
    }

    return result
}
```

</div>

> 💡 **注意**：Go 语言标准库没有内置的 `insert` 方法，因此需要手动实现插入逻辑（扩容 + 后移）。

## 📌 示例演示

以 `people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]` 为例：

1. **排序后**：

   ```
   [[7,0], [7,1], [6,1], [5,0], [5,2], [4,4]]
   ```

1. **逐个插入**：

   - 插入 `[7,0]` → `[[7,0]]`
   - 插入 `[7,1]` → `[[7,0], [7,1]]`
   - 插入 `[6,1]` → `[[7,0], [6,1], [7,1]]`
   - 插入 `[5,0]` → `[[5,0], [7,0], [6,1], [7,1]]`
   - 插入 `[5,2]` → `[[5,0], [7,0], [5,2], [6,1], [7,1]]`
   - 插入 `[4,4]` → `[[5,0], [7,0], [5,2], [6,1], [4,4], [7,1]]`

✅ 最终结果符合要求。

## ✅ 答案有效性证明

- **正确性**：

  - 由于我们按身高从高到低处理，当处理某人时，`result` 中已有的人都 ≥ 他；
  - 将其插入到第 `k` 位，意味着前面恰好有 `k` 个 ≥ 他身高的人；
  - 后续插入的更矮的人不会影响他的 `k` 值（因为矮的人不被计入“≥”）；
  - 因此最终每个人的 `k` 值都满足条件。

- **存在性**：

  - 题目保证输入合法，即存在至少一种有效排列；
  - 上述构造过程总能完成，不会出现越界（因为 `k ≤ 当前已插入人数`）。

## 📊 复杂度分析

| 项目 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 排序 | `O(n log n)` | `O(log n)`（排序栈空间） |
| 插入 | `O(n²)`（每次插入平均移动 `O(n)` 元素） | `O(n)`（结果列表） |
| **总计** | **`O(n²)`** | **`O(n)`** |

> 💡 虽然插入操作最坏是 `O(n²)`，但由于 `n ≤ 2000`（LeetCode 约束），实际运行效率可接受。

## 📌 问题总结

- **核心思想**：贪心策略 —— **先安排高个子，再安排矮个子**；
- **关键技巧**：排序规则（身高降序 + k 升序） + 按 k 插入；
- **适用场景**：涉及“相对顺序约束”的重建问题；
- **延伸思考**：若需支持动态插入/查询，可考虑使用 **Fenwick Tree（树状数组）** 或 **线段树** 优化插入位置查找至 `O(log n)`，使总复杂度降至 `O(n log n)`。

✅ 本题是贪心算法与排序结合的经典范例，体现了“局部最优导致全局最优”的设计哲学。
