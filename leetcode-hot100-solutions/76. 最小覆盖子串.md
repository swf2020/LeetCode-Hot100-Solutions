#  76. 最小覆盖子串

## 🔗 问题简介

[LeetCode 76. 最小覆盖子串](https://leetcode.cn/problems/minimum-window-substring?envType=problem-list-v2&envId=2cktkvj)

### 📝 题目描述

给你一个字符串 `s`、一个字符串 `t`。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果 `s` 中不存在涵盖 `t` 所有字符的子串，则返回空字符串 `""`。

**注意：**

- 对于 `t` 中重复字符，我们寻找的子字符串中该字符数量必须不少于 `t` 中该字符数量。
- 如果 `s` 中存在这样的子串，我们保证它是唯一的答案。

______________________________________________________________________

## 🧪 示例说明

**示例 1：**

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
```

**示例 2：**

```
输入：s = "a", t = "a"
输出："a"
解释：整个字符串 s 是最小覆盖子串。
```

**示例 3：**

```
输入：s = "a", t = "aa"
输出：""
解释：t 中两个 'a' 均应包含在 s 的子串中，
因此没有符合条件的子字符串，返回空字符串。
```

## 💡 解题思路

这是一个经典的 **滑动窗口（Sliding Window）** 问题。目标是在字符串 `s` 中找到一个最短的连续子串，使其包含 `t` 中的所有字符（包括重复字符）。

### ✅ 方法一：滑动窗口（双指针）

#### 步骤详解：

1. **统计目标字符频次**\
   使用哈希表（或数组）记录字符串 `t` 中每个字符出现的次数。

1. **初始化滑动窗口**\
   使用左右指针 `left` 和 `right` 表示当前窗口 `[left, right)`，初始都为 0。

1. **扩展右边界**\
   移动 `right` 指针，将字符加入窗口，并更新窗口内字符频次。\
   若某字符在窗口中的数量 ≥ `t` 中的数量，则认为该字符“已满足”。

1. **收缩左边界**\
   当窗口包含 `t` 中所有字符时（即“有效窗口”），尝试移动 `left` 指针缩小窗口，直到窗口不再有效。\
   在此过程中记录最小有效窗口。

1. **重复步骤 3–4**，直到 `right` 到达 `s` 末尾。

1. **返回结果**\
   若找到有效窗口，返回最短的那个；否则返回空字符串。

#### 关键点：

- 使用 `need` 记录 `t` 中各字符需求。
- 使用 `window` 记录当前窗口中各字符数量。
- 使用 `valid` 变量记录“已满足”的字符种类数（即 `window[c] >= need[c]` 的字符数）。

### 🔄 其他解法（简要）

| 方法 | 思路 | 时间复杂度 | 空间复杂度 | 适用性 |
|------|------|------------|------------|--------|
| 暴力枚举 | 枚举所有子串，检查是否覆盖 `t` | O(n² × m) | O(1) | ❌ 超时 |

## 💻 代码实现

```java:Java
class Solution {
    public String minWindow(String s, String t) {
        if (s == null || t == null || s.length() == 0 || t.length() == 0 || s.length() < t.length()) {
            return "";
        }

        int[] need = new int[128];
        int[] window = new int[128];
        
        // 统计 t 中字符频次
        for (char c : t.toCharArray()) {
            need[c]++;
        }
        
        int left = 0, right = 0;
        int valid = 0; // 满足条件的字符种类数
        int start = 0, len = Integer.MAX_VALUE;
        
        while (right < s.length()) {
            char c = s.charAt(right);
            right++;
            
            if (need[c] > 0) {
                window[c]++;
                if (window[c] == need[c]) {
                    valid++;
                }
            }
            
            // 尝试收缩左边界
            while (valid == countNonZero(need)) {
                if (right - left < len) {
                    start = left;
                    len = right - left;
                }
                
                char d = s.charAt(left);
                left++;
                
                if (need[d] > 0) {
                    if (window[d] == need[d]) {
                        valid--;
                    }
                    window[d]--;
                }
            }
        }
        
        return len == Integer.MAX_VALUE ? "" : s.substring(start, start + len);
    }
    
    private int countNonZero(int[] arr) {
        int count = 0;
        for (int x : arr) {
            if (x > 0) count++;
        }
        return count;
    }
}
```

```go:Go
func minWindow(s string, t string) string {
	if len(s) == 0 || len(t) == 0 || len(s) < len(t) {
		return ""
	}

	need := make(map[byte]int)
	window := make(map[byte]int)

	for i := 0; i < len(t); i++ {
		need[t[i]]++
	}

	left, right := 0, 0
	valid := 0
	start, minLen := 0, math.MaxInt32

	for right < len(s) {
		c := s[right]
		right++

		if _, ok := need[c]; ok {
			window[c]++
			if window[c] == need[c] {
				valid++
			}
		}

		for valid == len(need) {
			if right-left < minLen {
				start = left
				minLen = right - left
			}

			d := s[left]
			left++

			if _, ok := need[d]; ok {
				if window[d] == need[d] {
					valid--
				}
				window[d]--
			}
		}
	}

	if minLen == math.MaxInt32 {
		return ""
	}
	return s[start : start+minLen]
}
```

> - Go 版使用 `map[byte]int`，更通用（支持 Unicode），但稍慢。
> - 两者逻辑一致，均为标准滑动窗口。

## ✅ 答案有效性证明

1. **正确性**：

   - 滑动窗口确保所有可能的子串都被考虑。
   - `valid == len(need)` 保证窗口包含 `t` 中所有字符且数量足够。
   - 每次满足条件时立即尝试收缩，确保记录的是**当前右边界下最短**的有效窗口。

1. **最优性**：

   - 由于窗口只向右滑动（`left` 和 `right` 单调递增），不会遗漏更优解。
   - 所有有效窗口都会被比较，取长度最小者。

1. **边界处理**：

   - 空输入、`s` 比 `t` 短、无解等情况均被处理。

## 📌 问题总结

- **核心思想**：滑动窗口 + 哈希计数。
- **关键技巧**：
  - 用 `valid` 跟踪“已满足”的字符种类，避免每次遍历整个哈希表。
  - 先扩展右边界找可行解，再收缩左边界优化解。
- **适用场景**：子串匹配、覆盖问题、满足某种条件的最短/最长连续子数组。
- **易错点**：
  - 忘记处理 `t` 中重复字符。
  - 收缩窗口时未正确更新 `valid`。
  - 边界条件（如空字符串、无解）未处理。
